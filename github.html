Informasjon om GitHub

GitHub er et program som for det meste blir brukt til å samarbeide. Grunnen til dette er fordi det er mulig å legge ut alle filene på GitHub. Etter det kan alle som har tilgang til den mappen endre på filene. GitHub gir brukerne flere muligheter som er bra å ha under et prosjekt. Eksempler på dette er muligheten til å kunne gjenopprette tidligere versjoner av en fil. Du kan se på endringene andre ha gjort og sammenligne før du setter inn endringen i hovedfila.

Et eksempel på stegene i GitHub

1. Lag et respository, dette er felles mappen

2. Lag ulike branche slik at de som vil kan gjøre de endringene de ønsker

3. Se på de ulike endringene og sammenlign

4. Det er alltid minst en mester i en respository, den personen kan da dra endringene inn i hovedfilen

5. Bekreft at endringene skal bli gjort
